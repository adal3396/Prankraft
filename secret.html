🎬 Behind the Pranks

🔊 Sound Effects — No Audio Files

All sounds are generated in real-time using the Web Audio API. Oscillators create different waveforms (sine, square, triangle) at specific frequencies. The "error" sound is a descending frequency, and the "success" sound is an ascending arpeggio. Zero external files.

🎭 Prank Tracking — The Counter System

A central trackPrank(id) function manages all 21 pranks. It uses a Set to avoid duplicates, triggers a counter update with a CSS pulse animation, and auto-shows the reveal modal at 8 pranks. The counter is persistent per session.

💀 Fake BSOD — The Blue Screen

The BSOD is a full-screen overlay with a z-index: 9999. The percentage counter uses setInterval with randomized increments (3-12%) to feel "realistic." After completing, it plays a success sound and dismisses. The error code is PRANK_NOT_HANDLED.

🍪 Cookie Banner — 4 Stages of Rejection

The reject button tracks click count. Stage 1: banner moves to top. Stage 2: buttons change text. Stage 3: fake dismissal, then it returns ("I'm back 👋"). Stage 4: finally dismisses. The classList.add('hidden') ➝ setTimeout ➝ classList.remove('hidden') pattern creates the fake dismiss effect.

🌌 Particle Network — Interactive Background

60+ particles rendered on <canvas> with connection lines drawn between nearby particles. Mouse proximity creates a repulsion force: force = (1 - distance/MOUSE_RADIUS) * FORCE. Each particle has velocity, drift, and pulse effects. Total redraw happens at 60fps via requestAnimationFrame.

📑 Tab Title Prank — Visibility API

Uses document.addEventListener('visibilitychange') to detect tab switches. When hidden, a setInterval cycles through 7 funny messages. On return, it briefly shows "🎉 You're back!" before restoring the original title.
